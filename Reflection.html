(Note: For Q2, this page is using a CSS)

What did I learn? A lot actually. We've started with the basics and it becomes more complex in some ways.
Doing a basic HTML website is somewhat like an adventure, your long typed codes just to show a simple text or word honestly is very challenging.
And as you build it, continously add more, improve it, modify the codes to apply some new topics and seeing your website grow is exhilirating.
I will admit that the codes will actually be more and longer than the actual output that you'll see, and if you think about it, you'll understand how amazing the people who are actually doing this professionally.

Let's look back at Q1, I really liked how you can customize the fonts and colors, you can present yourself via this method.
You can expand and add more tweaks by adding tables and bullets for the aesthetics and presentation.
Adding emphasis on some texts to properly convey your messages, or adding links to provide sources.
Just remember that being colorful or adding different fonts and design doesn't necessarily mean it will look great, stay on the balanced and look at your output as an audience.

Q2 is when you expand on the basics, this is where you've realized that you can actually improve your website.
And when I say improve your website, I am talking about CSS. CSS is what makes coding a bit easier and make your website faster.
CSS can decrease the amount of code typing that you'll do since you can use one CSS source for multiple pages.
CSS provides a variety of customization coming from a specific source and adds those styles to several pages on your website.

This is when you realize something, is it better if we learned CSS first since we can code the styles on one CSS instead of each HTML pages?
I think no, I believe that building a great foundation is more important than being advanced.
You'll be trained to type styles repetatively which helps in the learning.
Discovering advanced coding like CSS is a perk, once you have a great foundation, you'll easily and properly utilize CSS, that is how I see it.

I will definitely admit though, there are a lot of codes, techniques and ways to create an HTML website.
I am still catching up, there is this point that I can only enter a code if I have something to look at or review from as a source.
I am currently learning, and still far from remembering everything.
I do try to practice, since practice makes you remember, and remembering becomes instinctive which results to mastery.

Reflecting on my weakness in coding has made me realize that I am still far from being good.
I would say that the time that I can say I am good with this is when the time I have stopped looking at my notes or resources.
Since I understand this weakness, it will now depend on my next action.
I will continue to learn, understand and practice, excited to see what comes from the next quarter.
